AUTHENTICATION FLOW:

Sign Up:
1. User selects role (customer/pharmacy/delivery)
2. Enters name, email, password
3. Firebase creates auth user
4. Firestore profile created
5. Backend /api/v1/auth/setup-profile called
6. Redirected to dashboard

Sign In:
1. User enters email/password or phone/OTP
2. Firebase validates credentials
3. Firestore profile fetched
4. AuthContext updated with user data
5. Redirected to dashboard

Dashboard Access:
1. Protected routes check auth state
2. Redirect to login if not authenticated
3. Load role-specific sidebar
4. Display role-specific content

API CALLS:

All API requests automatically include Firebase ID token:
Authorization: Bearer <id_token>

Example:
const data = await apiClient.get('/orders');
const created = await apiClient.post('/orders', { items: [] });
const updated = await apiClient.put('/orders/1', { status: 'delivered' });
await apiClient.delete('/orders/1');
